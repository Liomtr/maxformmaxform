### 2. 修补问卷访问控制

修改文件：

- `backend/src/routes/surveys.js`
- `backend/src/middlewares/auth.js`

主要改动：

- 新增 `optionalAuth` 中间件，用于公开访问场景下识别登录用户但不强制要求登录
- 新增问卷访问判定：
  - 已发布问卷可公开访问
  - 草稿/关闭问卷仅本人或管理员可访问
- 新增问卷管理判定：
  - 更新、删除、发布、关闭只允许问卷创建者或管理员执行
- `GET /api/surveys/:id` 支持通过 `id` 或 `share_code` 读取问卷
- `GET /api/surveys/share/:code` 同步支持“公开已发布 / 本人 / 管理员”访问规则
- `GET /api/surveys/:id/results` 改为必须登录且仅限本人或管理员查看
- 管理员获取问卷列表时不再被强制限制为“只能看自己创建的问卷”

收益：

- 修补了原先“任意已登录用户可修改/删除/发布/关闭他人问卷”的高风险漏洞
- 避免未发布问卷被公开读取
- 兼容前端当前 `/s/:id` 既可能传真实 ID、也可能传分享码的混合状态

### 3. 修补答卷访问控制

修改文件：

- `backend/src/routes/answers.js`

主要改动：

- 所有答卷读取、计数、导出、批量删除操作，均先校验对应问卷是否归当前用户管理
- 非管理员查询答卷列表时，必须提供 `survey_id`
- 答卷详情访问会先反查所属问卷，再校验权限
- 批量删除前会逐条校验所选答卷所属问卷权限
- Excel 导出仅允许问卷所有者或管理员执行

收益：

- 修补了“任意登录用户可能查看、导出、删除他人答卷”的高风险问题

### 4. 修补文件访问控制

修改文件：

- `backend/src/routes/files.js`

主要改动：

- 普通用户文件列表只返回自己上传的文件
- 管理员可按 `uploader_id` 查看指定用户文件

收益：

- 修补了“任意登录用户查看全站文件”和“匿名上传图片”的高风险问题

### 7. 打通问卷设置保存与执行

修改文件：

- `backend/src/routes/surveys.js`
- `backend/src/models/Answer.js`
- `frontend/src/views/survey/CreateSurveyPage.vue`
- `frontend/src/views/survey/FillSurveyPage.vue`
- `frontend/src/views/survey/FillSurveyMobilePage.vue`

主要改动：

- 编辑器保存问卷时，开始把 `settings` 一并提交到后端
- 编辑器加载已有问卷时，开始回填：
  - `allowMultipleSubmissions`
  - `endTime`
- 发布问卷时增加截止时间校验：
  - 若 `settings.endTime` 早于当前时间，则禁止发布
- 公开访问填写页时增加截止时间校验：
  - 已截止问卷返回 `403`
  - 返回包含标题、截止时间、状态的元信息
- 提交答卷时增加截止时间校验：
  - 已截止问卷拒绝提交
- 提交答卷时增加重复提交限制：
  - 若 `allowMultipleSubmissions !== true`，则默认禁止重复提交
  - 当前实现按 `IP + survey_id` 判重
- 补充 `Answer.countByIp()`，作为重复提交限制的底层查询
- 填写页错误提取逻辑开始兼容后端 `error.message`

当前边界：

- 重复提交限制目前是“按 IP 防重”，不是账号级、防设备级或强实名级
- 该策略适合先把规则真正落地，不适合作为最终严格风控方案

收益：

- 解决了“前端能配置截止时间/重复提交限制，但后端不执行”的问题
- 让问卷设置从“展示字段”变成“真实生效规则”

### 8. 让填写页真正响应“进度显示 / 随机题序”设置

修改文件：

- `frontend/src/views/survey/FillSurveyPage.vue`

  - 移动端填写页显示当前作答进度
- 当问卷启用 `randomizeQuestions` 或 `randomOrder` 时：
  - 进入填写页时对题目顺序进行一次随机化
  - 桌面端和移动端都生效
- 编辑器再次打开已有问卷时，开始回填：
  - `randomizeQuestions`
  - `collectIP`
  - `submitOnce`
- 前端 `SurveySettings` 类型补齐：
  - `randomizeQuestions`
  - `collectIP`

当前边界：

- 题目随机化目前是“进入页面时随机一次”，不是按页刷新稳定复现或按用户会话持久化复现
- 进度统计基于“当前可见题目中已填写数量”，不含隐藏题

收益：

- 填写页开始真正体现设置项效果，而不只是保存配置
- 桌面端与移动端在设置行为上更一致
