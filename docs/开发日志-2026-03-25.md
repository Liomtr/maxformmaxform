### 2. 处理上传题正向提交流程的脚本口径

本轮联调中暴露出一个实现细节：

- 上传题在提交答卷前，会先经过当前的答案级 `accept` 校验。
- 如果答案里只传 `id / uploadToken`，而不携带文件 `name / type / size`，就可能先在原始值校验阶段被判成“不支持的文件类型”。

本次处理方式：

- 系统冒烟中的正向上传题提交流程，显式补齐：
  - `name`
  - `type`
  - `size`
  - `id`
  - `uploadToken`

当前判断：

- 这是当前服务端校验顺序下的实际输入契约，脚本已按现状对齐。
- 后续若要进一步收敛前端负担，可考虑在服务端先解析 file id，再做类型校验。
