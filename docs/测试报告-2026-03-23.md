## 5. 本轮新增发现与处理

在新增 `frontend/package.json` 脚本后，首次执行 `npm run smoke:system` 暴露出一个初始化问题：

- `scripts/system-smoke.mjs` 只在本地变量中构造了默认数据库环境
- 但 `backend` 的数据库连接模块在导入时就读取 `process.env`
- 导致从 `frontend` 目录执行 npm 脚本时，数据库连接拿到的是空密码，出现 `Access denied for user 'root'@'localhost' (using password: NO)`

已处理结果：

- 调整 `scripts/system-smoke.mjs` 初始化顺序
- 先将默认环境值写入 `process.env`
- 再动态加载 `migrate`、`knex`、`User`、`Role`
- 修复后 `cmd /c npm run smoke:system` 已实测通过
