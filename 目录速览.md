# 目录速览

本文档只按当前仓库源码描述，不沿用旧的 MongoDB / ClickHouse / `app.cjs` / `simple-server.js` 方案。

## 一句话理解

- `backend/`：`Express + Knex + MySQL` 后端，问卷域已进入模块化单体分层阶段。
- `frontend/`：`Vue 3 + TypeScript + Vite` 单页应用，已拆出后台管理、问卷编辑、填写、结果分析等视图。
- `shared/`：前后端共享的题型注册表与题目统计模型，是问卷领域的单一事实源。

## 根目录

- `backend/`：后端服务。
- `frontend/`：前端工程。
- `shared/`：题型元数据与统计口径共享层。
- `scripts/`：系统冒烟、清理等辅助脚本。
- `docs/`：项目文档与阶段性评估。
- `README0326.md`：当前项目总览与启动说明。
- `后端-启动.cmd`、`后端-生产-启动.cmd`、`前端-启动.cmd`、`前端-生产-预览.cmd`：根目录启动脚本。

## backend/

当前后端真实入口和结构如下：

- `server.js`：实际启动入口。先检查 MySQL 连通性，再启动 HTTP 服务。
- `app.js`：Express 应用装配层，挂载中间件、静态资源、路由和错误处理。
- `initAdmin.js`：初始化管理员账号。
- `scripts/start-dev.js`、`scripts/start-prod.js`：开发/生产启动包装脚本。
- `src/config/`：环境变量与运行配置。
- `src/db/`：Knex 连接、迁移、种子、事务封装。
- `src/models/`：面向表的访问层，负责 CRUD 与兼容字段映射。
- `src/routes/`：HTTP 路由入口。
- `src/policies/`：访问控制与问卷策略判断。
- `src/services/`：问卷域业务服务，已拆为 query / command / upload / results。
- `src/repositories/`：聚合写入仓储，负责事务边界内的多表协同。
- `src/middlewares/`：鉴权与错误处理中间件。
- `src/utils/`：题型校验、上传存储等工具。
- `test/`：Node `node:test` 路由与服务测试。

### 后端当前调用链

问卷域当前更接近下面这条链路：

```text
HTTP Request
  -> route
    -> policy / access check
      -> service
        -> repository + transaction
          -> model
            -> JSON response
```

说明：

- `src/routes/surveys.js` 已主要承担参数提取和响应封装。
- `src/services/surveyQueryService.js`、`surveyCommandService.js`、`surveyUploadService.js`、`surveyResultsService.js` 分别负责读、写、上传、统计。
- `src/repositories/surveyAggregateRepository.js` 承担答卷提交、清空回收站、彻底删除等聚合写操作。
- `src/routes/answers.js`、`src/routes/users.js` 等模块仍偏向 `route -> model`，尚未完全复制问卷域的拆分方式。

## frontend/

前端当前真实结构如下：

- `src/main.ts`、`src/App.vue`：应用入口。
- `src/router/index.ts`：路由定义与前置守卫。
- `src/layouts/`：后台布局壳。
- `src/stores/`：Pinia 状态管理，目前主要是认证态。
- `src/api/`：接口封装。
- `src/views/admin/`：后台管理页面。
- `src/views/survey/`：问卷创建、编辑、填写、结果、答卷管理。
- `src/composables/`：跨页面组合式逻辑。
- `src/composables/editor/`：问卷编辑器按能力拆分后的子 composable。
- `src/components/`：通用组件。
- `src/mappers/`、`src/types/`：前端数据映射与类型定义。
- `src/utils/`：题型注册、导入、图表、上传等工具。
- `src/styles/`：全局样式与设计令牌。

### 问卷编辑器当前调用链

```text
route
  -> CreateSurveyPage / FillSurveyPage
    -> CreateSurveyPageShell / page components
      -> useSurveyEditor / useFillSurvey*
        -> composables/editor/*
        -> api/*
          -> backend /api/*
```

说明：

- `CreateSurveyPage.vue` 已是薄入口。
- `useSurveyEditor.ts` 现为装配层，聚合 `editor-core`、`editor-logic`、`editor-batch`、`editor-quota`、`editor-richtext`。
- 填写页逻辑已拆成 `useFillSurveyVisibility.ts`、`useFillSurveyJumpLogic.ts`、`useFillSurveyQuota.ts`、`useFillSurveyUpload.ts`。

## shared/

- `shared/questionTypeRegistry.js`：题型定义、legacy `uiType`、渲染类型、统计类型、提交语义。
- `shared/questionModel.js`：题目级统计聚合逻辑。

这是当前问卷题型语义和统计口径的统一入口。前端编辑器、后端校验、结果统计都依赖这层。

## 运行与联调

- 后端默认端口：`63002`
- 前端默认端口：`63000`
- 健康检查：`GET /health`
- 前端开发代理：
  - `/api` -> `http://127.0.0.1:63002`
  - `/uploads` -> `http://127.0.0.1:63002`

注意：

- 当前启动后端不会自动执行 `migrate()` 或 `seed()`。
- 首次启动前应手动执行：
  - `cd backend`
  - `npm run db:migrate`
  - `npm run db:seed:dev`
- 系统冒烟脚本位于 `scripts/system-smoke.mjs`，会直接验证后端主链路。
